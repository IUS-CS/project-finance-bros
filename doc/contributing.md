# Contributing

### AI Policy

Our group will not be consulting AI or generating code with AI for this project. While it could and probably would make creating this site significantly easier, we wanted to prioritize a deeper understanding of the technologies at hand. By relying on AI, we may have a more feature rich product but not fully understand the design decisions of the code the AI is pulling from. By making and learning from our own mistakes, we should be able to make better apps in the future. Even if AI generates the code for these future projects.

### Code Review Process
The reviewer will pull the branch on their own machine and run the added code, when applicable, in order to test before merging. We figured that having the code on your own machine would make detecting errors easier. We had a situation in our first sprint where a directory was misnamed/duplicated but this wasn't seen on github's file explorer. Pulling to the machine is pulling to an environment that the developer is comfortable with and gives the best chance of catching mistakes. The requirement of running the code ensures that the addition was seen by another human and that the addition works well on multiple machines.
